# nginx architecture — v1.2 (nginx-only authentication)

This document captures the v1.2 rewrite and why it exists. Read
`nginx/nginx.conf` alongside this file — the config is the
authoritative artefact; this page is the *why*.

## The invariant

> nginx is the **sole** authentication and authorization boundary.
> FastAPI is auth-blind.

Any request that reaches the Python process has already:

1. Completed an mTLS handshake against `pki/nginx/nginx.crt`.
2. Had its client cert validated against `pki/ca/ca.crt`.
3. Had its client cert checked against `pki/ca/ca.crl`.
4. Had its Subject CN looked up in the allowlist (`map $ssl_client_cn
   $cn_allowed`) and found to be admitted.

If any of those gates fails, nginx returns a terminal status directly
(400 / 403 / TLS handshake abort) and the upstream is never contacted.

## How we got here — v1.0 → v1.1 → v1.2

| Version | TLS termination | CN allowlist              | Coupling |
|---------|-----------------|---------------------------|----------|
| v1.0    | FastAPI (stdlib ssl) | FastAPI middleware    | FastAPI owns both |
| v1.1    | nginx                | FastAPI middleware     | Hybrid — nginx forwards cert fields; FastAPI re-parses |
| v1.2    | nginx                | nginx (`map{}` + `if`) | nginx owns both; FastAPI is auth-blind |

The v1.1 hybrid shipped working code but carried two structural
problems: (a) two places to change when the allowlist moves — nginx
and `config.ALLOWED_CLIENT_CNS` — and (b) a non-trivial "trusted
proxy IP" check in the FastAPI middleware to stop anyone bypassing
nginx by talking to FastAPI directly. Both disappear in v1.2.

## The allowlist lives in nginx.conf

```nginx
map $ssl_client_s_dn $ssl_client_cn {
    ~(?:^|,)\s*CN=(?<cn>[^,]+) $cn;
    default                    "";
}

map $ssl_client_cn $cn_allowed {
    default       0;
    "client-01"   1;
    "client-02"   1;
}
```

To admit a new client:

1. Add its CN to the second `map`.
2. `make nginx-reload` (or `nginx -s reload`).

There is no Python code to edit. There is no restart of FastAPI.
Structural test `ST2` in `tests/test_v12_structural.py` (added in
N3v2) fails CI if anyone reintroduces an allowlist in `config.py`.

## The 403 body is emitted by nginx

```nginx
location / {
    if ($cn_allowed = 0) {
        add_header Content-Type application/json always;
        return 403 '{"error":"forbidden","cn":"$ssl_client_cn","reason":"cn_not_allowlisted"}';
    }
    proxy_pass http://fastapi_backend;
    ...
}
```

FastAPI never sees a denied request. The shape of the JSON matches the
v1.1 body byte-for-byte so existing clients need no changes.

## TLS layer & CRL

`nginx/ssl_params.conf` pins the TLS 1.2+ floor and a `HIGH` cipher
list without RC4/3DES/EXPORT/DES/MD5. The client side is verified
against `pki/ca/ca.crt` with `ssl_verify_client on` and
`ssl_verify_depth 2`. `ssl_crl` is pointed at `pki/ca/ca.crl` — the
file is regenerated by `tests/revoke_client.sh` and a
`make nginx-reload` propagates a fresh CRL within milliseconds.

## FastAPI surface

`server.py` is a vanilla uvicorn process:

* No `ssl.SSLContext`.
* No `ssl_keyfile` / `ssl_certfile`.
* No cert-aware `H11Protocol` subclass.
* No peer-cert parsing, no `X-Client-*` header trust, no allowlist.

The single surviving middleware is a request-id + access-log shim
with **zero** authorization logic. Whether a request came via nginx
or straight to 127.0.0.1:8443 makes no difference to the Python
code — which is exactly the point: there is nothing to bypass,
because nothing is there to protect.

## Header trust model (spoiler: there isn't one)

Any `X-Client-*` header nginx forwards is purely for observability
(audit logs, debug traces). FastAPI MUST NOT make decisions based on
these headers. The N3v2 structural test suite (`ST3`) greps
`server.py` for `X-Client-CN` / `ssl` imports / `X509` references and
fails CI if any appear.

## Running the stack locally

```bash
make pki            # CA + server + nginx + client certs
make stack          # FastAPI plain HTTP :8443 + nginx mTLS :8444
curl --cacert pki/ca/ca.crt \
     --cert   pki/client/client.crt \
     --key    pki/client/client.key \
     https://localhost:8444/health     # 200 "ok" — allowlist says yes
make nginx-stop && make stop
```

To exercise the deny path, comment out the `"client-01"` line in the
`$cn_allowed` map and `make nginx-reload` — the same curl call then
returns the JSON 403 directly from nginx without touching FastAPI.
